# Changelog

All notable changes to `laravel-action` will be documented in this file.

## Unreleased

**Breaking changes**

- `PendingAction::run()` and `Testable::run()` were renamed to `then()`. Behavior is unchanged: the closure receives the wrapped action with full typing, the invocation runs through the whole configured chain (a cached `idempotent()` hit still returns without the closure executing), and the closure's result is returned. The old name suggested the method itself executed the action and said nothing about the pass-through; `then()` follows `Pipeline::then()`, which runs the pipes with the given closure as the innermost destination. There is no `run()` alias — replace `->run(fn (MyAction $a) => ...)` with `->then(fn (MyAction $a) => ...)`. The `memoize()` missing-key exception message names `then()` accordingly.
- A consumed `once()` key now routes through a chained `fallback()`: the fallback closure receives a new `OnceConsumedException` (carrying `key()`) and its value becomes the result — previously the skip answered `null` even with a fallback chained. Rethrow from the fallback to decline and the exception reaches the caller. Chains without a `fallback()` are unchanged: a skip still answers `null`. `once()` + `fallback()` now guarantees a result, which the PHPStan rules shipped in this release can verify statically.

## 2.0.0 - 2026-07-18

Idempotency, retries, circuit breakers and more.

**New features**

- Enum-backed events: declare `#[EmitsEvents(OrderEvent::class)]` (all cases become events) or mix cases into the array form, and pass cases anywhere an event name goes — `$this->event(OrderEvent::Placed, $data)`, `->on(OrderEvent::Placed, ...)`, `->forwardEvents([OrderEvent::Placed])`. String-backed enums use their value as the event name, pure enums their case name, and an enum case is interchangeable with its normalized string everywhere (a string listener receives enum-emitted events and vice versa). Int-backed enums are rejected with a clear exception. Plain string events keep working unchanged.
- New `events()` test instrument: `MyAction::test()->events(fn (Collection $events) => ...)->handle(...)` records the events the action under test emits (or, given class names, the events of nested actions resolved during the run), as `EmittedEvent` records with the normalized event name, payload and emitting action class. `EmittedEvent::is()` compares against strings and enum cases alike. Follows the same registrar shapes as `profile()`/`queries()`/`logs()` and routes through an overridable `addEvents()` hook.
- First-class N+1 detection: chain `->assertNoDuplicateQueries()` and/or `->assertQueryCount($n)` before `->handle(...)` — query recording is enabled automatically and the checks run once the action completes, failing the test with the offending SQL. Duplicates are grouped by connection and normalized SQL (whitespace collapsed, `in (?, ?, ?)` placeholder lists reduced), so N+1 loops and `whereIn` retries with different item counts are caught. Coexists with a `queries()` inspection callback, and — like all inspection callbacks — does not run when an `idempotent()` cache hit means nothing executed.
- Execution modes: `$action->memoize($key = null)` is per-process idempotency — the first successful result per key is remembered in a container-scoped store (flushed for free under Octane and between tests, or explicitly with `Action::flushMemoized()`) and later calls return it without executing; keys derive from the `handle()` arguments (pass one explicitly for unserializable arguments or with `run()`) and are scoped per action class. `$action->defer(fn (MyAction $a) => $a->handle(...))` runs the whole configured wrapper chain after the response has been sent, via Laravel's `defer()`.
- Lifecycle events: every wrapper-mediated invocation dispatches `ActionStarted`, then `ActionCompleted` (result, duration in ms, memory delta) or `ActionFailed` (exception, duration, memory delta) as class-based Laravel events — one `Event::listen(ActionCompleted::class, ...)` turns every wrapped action into an APM data point. A bare `$action->observed()` opts a plain call in; direct `handle()` calls cannot emit (the base `Action` has no interception point around a user's `handle()`).
- `MyAction::test()->dryRun()` runs the action inside database transactions that are rolled back afterwards: the instruments still record and report, the result is still returned, only the database work is discarded — "what would this action do". Composes with `idempotent()`: a key persisted during a dry run is discarded with the rollback. Side effects outside the database are not contained.
- Concurrency and transaction wrappers on the same chain: `$action->withoutOverlapping($key, $wait, $staleAfter, $store)` gives mutex semantics per key — every call runs eventually, never two at once; a held lock fails immediately (or after waiting `$wait` seconds) with a `LockTimeoutException`, and lock-less cache stores are rejected loudly. `$action->throttle($key, $allow, $every)` rate-limits executions per key through Laravel's `RateLimiter`, throwing a `ThrottledException` (carrying `availableIn()`) once the budget is exhausted — deliberately retryable, so `retry(backoff:)` around it waits a window out while every attempt inside `retry()` consumes budget. `$action->transactional($attempts, $connection)` wraps the run in `DB::transaction()` with deadlock-retry attempts, nested innermost so every retry attempt gets a fresh transaction. Keys default to the action class.
- Resilience wrappers, chainable with `idempotent()` on one wrapper (`PendingAction`): `$action->retry($times, $backoff, $when)` re-runs a failing action with a fixed, per-attempt or closure-computed backoff (slept through `Sleep`, so tests fake it) — the `NonRetryable` marker interface opts exceptions out of retrying and an explicit `$when` filter overrules it; `$action->fallback(fn (Throwable $e) => ...)` answers with a degraded value when the action ultimately throws; `$action->circuitBreaker($key, $threshold, $cooldown, $store)` fails fast with a `CircuitOpenException` (carrying `availableIn()`) after too many consecutive failures and lets a single lock-guarded probe through once the cooldown passes. The nesting order is fixed regardless of chaining order — fallback → idempotent → retry → circuit breaker — so failed attempts never consume an idempotency key, every attempt feeds the breaker, an open circuit is not retried, and a fallback value is never cached as a real result.
- Idempotent execution: `$action->idempotent($key, $ttl = null, $store = null)->handle(...)` runs the action at most once per key and returns the cached result on later calls. Keys are scoped per action class, results are stored in an envelope so a `null`/`false`/`''` result still counts as executed, only successful runs consume the key (a thrown exception leaves it free to retry), and concurrent callers are serialised with a cache lock when the store supports one. Bust an entry with `$action->forgetIdempotency($key, $store = null)`. The wrapper mirrors the action's own `handle()` signature via a generic `@mixin` (typed arguments and inferred return in PHPStan); `run(fn (MyAction $a) => $a->handle(...))` offers the same typing through a closure for editors that don't resolve generic mixins yet. Chainable with the test instruments — `MyAction::test()->profile(...)->idempotent($key)->handle(...)` — sharing keys with the production wrapper; on a cache hit nothing executes, nothing is instrumented and no callbacks fire (`wasExecuted()` tells the runs apart). Inside an open database transaction on the default connection, the idempotency key is only consumed once the transaction commits — rolled-back work leaves the key free.
- Execution trace: chain `->trace()` on any wrapped action to record a decision-by-decision timeline — retry attempts and sleeps, idempotency hits/stores, memoize hits, acquired overlap locks, circuit-breaker trips/probes/rejections, throttle rejections, transaction commits/concurrency-retries — read back via `lastTrace()`, an optional `trace(fn (Trace $trace) => ...)` callback (also called when the run throws), or the lifecycle events. `Trace` offers `entries()`, `has()`, `count()`, `first()`, `durationMs()` and a human-readable `summary()`; `dumpTrace()`/`ddTrace()` print it while developing. Tracing costs nothing unless enabled.
- Automatic log context: every wrapper-mediated run and every `test()` run sets the running action's class under the `action` key in Laravel's Context, so log lines written during the run carry their action. Innermost-action attribution for nested runs, exception-safe restore, pre-existing values survive.
- `retry(..., jitter: true)` sleeps a random duration between zero and the scheduled backoff instead of the exact value, spreading synchronized retry waves ("thundering herd") out. Composes with all backoff shapes.
- `when()`/`unless()` (Laravel's Conditionable) on `Action`, `PendingAction` and `Testable`: `SendInvoice::make()->when($cond, fn (SendInvoice $a) => $a->throttle())->handle($order)`.
- `assertExecuted()`/`assertSkipped()` on `Testable`: post-hoc assertions turning `wasExecuted()` into proper test failures, with a distinct message when `idempotent()` was never configured or `handle()` has not run.
- Per-instrument dump helpers on `Testable`: `dumpQueries()`/`ddQueries()`, `dumpLogs()`/`ddLogs()`, `dumpEvents()`/`ddEvents()`, `dumpProfile()`/`ddProfile()` — chain before `handle()`, recording auto-enables, a formatted report prints after the run (`dd*` stops the process, like `DB::ddRawSql()`). Skipped on `idempotent()` cache hits like all inspection callbacks.
- `$action->once($key, $ttl, $store)` (and `Inline::once()`) runs `handle()` at most once per key, keeping nothing but the key: the first successful run stores a bare `true` under the verbatim cache key and every later call is skipped, answering `null` — `idempotent()` without the cached result, for side-effect actions whose return value isn't worth replaying. Any existing cache entry under the key counts as consumed, whoever wrote it, so keys written by other systems are respected. Only successful runs consume the key, the write is transaction-aware (deferred to commit), concurrent callers are serialised through a lock when the store supports one, `wasExecuted()` distinguishes a run from a skip, and traces record the decision (`once: consumed` / `once: skipped`).
- Inline actions: `Inline::idempotent('key')->retry(3)->handle(fn () => ...)` runs a closure through the whole execution pipeline — every wrapper, tracing, lifecycle events, log context — without declaring an action class. Real static entry points keep full IDE and PHPStan typing (each returns the same `PendingAction` the class-based API uses). The closure receives the underlying action, `Inline::events([...])` declares emittable events fluently (the attribute has no class to sit on), and `on()` now exists on `PendingAction` itself so listening mid-chain no longer derails onto the action and silently drops configured wrappers. Because all inline actions share one class, the class-key-scoped wrappers (`circuitBreaker()`, `throttle()`, `withoutOverlapping()`, `memoize()`) require an explicit key on inline actions and throw a clear `InvalidArgumentException` otherwise.
- `php artisan make:action ShipOrder` scaffolds an action class — default directory `app/Actions`, `--dir=` overrides it and the namespace follows the directory (`app/Domain/Orders` → `App\Domain\Orders`), nested names (`Orders/ShipOrder`) create subdirectories and subnamespaces, `--force` overwrites. `--events` also generates a companion string-backed `ShipOrderEvent` enum next to the action, wired through `#[EmitsEvents(ShipOrderEvent::class)]`. Customize the generated code by placing `action.stub`, `action.events.stub` or `action-event.stub` in your application's `stubs/` directory.

**Breaking changes**

- Idempotency keys are no longer namespaced: `idempotent()` uses the given key verbatim as the cache key instead of prefixing it with the action class. Entries written by earlier versions are never read again (at worst an action runs once more; stale entries expire per their TTL), two actions given the same key now share the entry, and `forgetIdempotency($key)` forgets exactly `$key`. The `@internal` `Idempotency::keyFor()` helper was removed.
- `illuminate/support` is now required at `^11.23` (up from `^11.0`) for Laravel's `defer()`.
- `Testable::handle()` is no longer a declared `handle(mixed ...$args)` method: like the production wrapper (`PendingAction`), the class-level generic `@mixin` now projects the wrapped action's own `handle()` signature (typed arguments and return in PHPStan) and `__call` intercepts the invocation, forwarding any other unknown method to the wrapped action. Runtime behavior is unchanged. Editors that do not resolve generic mixins yet can use the new `Testable::run(fn (MyAction $a) => $a->handle(...))`, which runs the closure through the full instrumented flow (dry run and idempotency included) with checked arguments and an inferred return type.
- The mock-binding test helpers — `Action::fake()` and `Testable::only()`/`without()`/`except()` — now throw a `RuntimeException` when used outside a test context (`runningUnitTests()` or the `local`/`testing` environments), instead of silently binding mocks into the container. A forgotten helper in staging/production previously replaced real actions with mocks (worst under Octane). To profile or debug in production, use the ungated observability helpers (`test()->profile()/queries()/logs()`); to deliberately bind mocks outside a test environment, opt in with `Action::allowTestHelpers()`.
- `Profile::memoryUsed()`, `startMemory()`, `endMemory()` and `peakMemory()` no longer take a unit argument and now return a `MemorySize` value object. Use `->bytes()`, `->in('KB')` or `->format()` on the result. This also fixes fractional unit conversions being silently truncated to integers.
- `Memory::formattedMemory()` was replaced by `Memory::size()`, returning a `MemorySize`.
- `MemoryFormatter` was replaced by the `MemorySize` value object.
- `only()`, `without()`, `except()`, `profile()`, `queries()` and `logs()` now reject container aliases and non-action classes with a clear exception instead of failing later with a fatal error.
- Event forwarding now resolves its ancestors when `forwardEvents()` is called instead of on every `event()` emission. Call `forwardEvents()` from within the scope that should receive the events — the idiomatic `Action::make()->forwardEvents()->handle()` chain is unaffected, but an action configured in one scope and executed in another now propagates to the ancestors that enclosed the `forwardEvents()` call (calling it again re-captures), no longer to whatever encloses `handle()`. Captured ancestors are held weakly, so forwarding never keeps a service alive, and emitting events no longer walks the call stack.
- `ActionCompleted` and `ActionFailed` gained a trailing `?Trace $trace` constructor parameter (the execution trace when tracing was enabled, null otherwise). Listeners are unaffected; code constructing these events manually must pass the extra argument.
- The `@internal` `Iak\Action\Execution\Middleware` interface gained `traceTo(TraceRecorder $recorder): void` (implemented for all bundled middleware by the `TracksTrace` trait). The middleware set is curated by the `PendingAction` API and was never a public extension point.
- `Testable::on()` now keeps the chain — it returns the `Testable`, mirroring `PendingAction::on()` — instead of forwarding to the action and returning that, which made anything chained after it bypass the instruments. And chaining an execution wrapper (`retry()`, `fallback()`, `circuitBreaker()`, `throttle()`, `withoutOverlapping()`, `transactional()`, `memoize()`, `observed()`, `trace()`/`dumpTrace()`/`ddTrace()`, `defer()`) onto a `Testable` now throws a `LogicException` instead of silently returning the production wrapper — which dropped every configured instrument, dry run and idempotency, so inspection callbacks never ran and their assertions passed vacuously. Configure execution wrappers on `MyAction::make()`; `idempotent()` remains supported on `test()`.

**Improvements**

- Event propagation no longer captures call-stack arguments, and the per-call trait and attribute reflection (`usesHandlesEvents()`, `getAllowedEvents()`) is now cached per class.
- PHPStan level 9 (up from 6) with full generics: `Action::test()` returns `Testable<static>`, `Action::fake()` returns `static&MockInterface`, and all inspection callbacks have typed closure signatures, so editors autocomplete `$queries`, `$logs` and `$profiles` inside callbacks.
- `only()` now also mocks constructor-injected child actions, not just actions resolved during `handle()`.
- Auto-mocked actions return a zero value matching their declared `handle()` return type (`''`, `0`, `false`, `[]`) instead of `null`.
- Declared the runtime dependencies (`illuminate/support`, `illuminate/database`, `monolog/monolog`, `nesbot/carbon`) and suggested `mockery/mockery` for the testing helpers.
- The `profile()`/`queries()`/`logs()` test instruments now share one generic `Instrumentation` descriptor internally instead of three copies of the registration, interception and collection machinery — groundwork for future instruments.
- Every collection path — nested-action proxies and the action under test alike — now routes through the overridable `addProfile()`/`addQueries()`/`addLogs()` hooks, so a `Testable` subclass overriding them sees all collected results. A listener mispaired with its instrumentation descriptor now throws a `LogicException` instead of silently reading no results.

**Bug fixes**

- Falsy mocked return values (`false`, `0`, `''`) passed to `without()` are no longer silently converted to `null`.
- Fixed a fatal error when profiling or auto-mocking actions whose `handle()` declares a return type.
- Fixed a crash in the event-cleanup destructor when it fired between tests while facades pointed at a flushed application.
- Container bindings replaced by `profile()`/`queries()`/`logs()` proxies are restored after `handle()`, and the `only()` hook deactivates itself instead of intercepting resolutions (and leaking the `Testable`) forever.
- `profile()`/`queries()`/`logs()` now reject `final` action classes with a clear exception instead of an uncatchable fatal when the proxy is created.
- The `final`-class guard also applies to `queries($callback)`: that path additionally registers the action under test as a proxy target, so a `final` self-resolving action previously fataled when re-resolved mid-run instead of failing at registration.
- `ProfileListener` removes its `action.record_memory.*` listeners after the profiled run instead of leaking them.
- `forwardEvents(['event-a'])` now forwards only the named events, as documented: previously any non-empty list enabled forwarding wholesale, so events excluded from the list still propagated to an ancestor that allowed them.
- Ancestor detection for event forwarding now sees `HandlesEvents` composed through an intermediate trait; previously a class using its own trait that in turn uses `HandlesEvents` silently never received forwarded events.
- `dumpQueries()`/`ddQueries()` now report query times in milliseconds as labeled; previously they printed the stored seconds value with an `ms` suffix, understating every duration by 1000×. (`Query::$time` is documented as seconds now — use `Query::duration()` for a unit-safe interval.)
- `QueryListener` is no longer pinned for the container's lifetime (with its recorded queries) by the closure it registers through `DB::listen()`: the closure now holds a weak reference, so a finished listener is garbage-collectable. Previously every instrumented run leaked one — which mattered on the supported profile-in-production path under Octane, where the container survives across requests.
